Quick Start

Reference DIS schemas directly in your JSON files for instant IDE autocomplete and validation:

{ "$schema": "https://schemas.domainintelligenceschema.org/dis/1.6.0/DISDossier.schema.json", "dossierId": "your-dossier-id", "name": "Customer Service Domain", "disSpecificationRef": "1.6.0" }

Validation with AJV

ajv validate \ -s https://schemas.domainintelligenceschema.org/dis/1.6.0/DISDossier.schema.json \ -d my-dossier.json

TypeScript Type Generation

json-schema-to-typescript \ https://schemas.domainintelligenceschema.org/dis/1.6.0/Entity.schema.json \ > types/Entity.ts

Architecture Overview

DIS v1.6 is organized into six logical categories that build upon each other:

  1. Foundation ‚Äî Root structures and centralized vocabularies
  2. Identity & Structure ‚Äî Entities, roles, applications, and endpoints
  3. Behavioral Grammar ‚Äî AgenticTriplets (Entity-Mode-Role) and mode definitions
  4. Matrices ‚Äî Execution bindings, RBAC rules, capabilities, and relationships
  5. Supporting Constructs ‚Äî Reusable functions, knowledge docs, tags, and validation rules
  6. Business Value & Governance ‚Äî Change management, privacy, telemetry, marketplace, and ROI tracking

What's New in v1.6

  • CommonEnums: Centralized DRY-compliant enum definitions (31 enums)
  • Vendor Neutrality: Removed all proprietary references (e.g., AmeliaBotInstance)
  • JMESPath Standard: Unified expression language for queries and transformations
  • 6 New Governance Constructs: Enterprise-ready change management, privacy, telemetry, marketplace, comparison, and value engineering
  • Enhanced Documentation: Every construct includes comprehensive descriptions and examples
